# LLM Wiki 知识库管理 Agent

> 🛠️ 用 AI Agent 自动维护 Markdown 知识库（添加、搜索、更新、整理）

| 属性 | 信息 |
|------|------|
| 类型 | 自动化 / 知识管理 |
| 适用工具 | Claude Code / Codex |
| 依赖 | Git 仓库 + Markdown 目录 |

## 功能说明

这个 Skill 参考 Karpathy 提出的 LLM Wiki 思路，让 Agent 以“知识节点 + 链接关系”的方式维护个人知识库。

## Skill 内容

```markdown
<!-- TODO: 填入完整的 Skill prompt -->
你是一个知识库维护 Agent。
目标：持续维护 docs/ 目录下的 Markdown 文档。
行为：
1. 新增知识时，先检查是否已有同主题页面。
2. 修改内容时，保持原有结构与链接有效。
3. 每次更新后输出“新增/修改/删除”清单。
4. 对低置信信息标注 TODO，避免伪造事实。
```

## 使用方法

1. 指定知识库根目录。
2. 提供新增资料或更新需求。
3. 让 Agent 先生成变更计划，再执行修改。
4. 执行后检查 diff 并确认提交。

## 适用场景

- 个人学习笔记持续沉淀
- 项目文档自动维护
- 多来源资料的结构化归档

!!! warning "注意事项"
    - 重要页面建议保留人工审核流程
    - 避免让 Agent 在无约束下批量重写全部文档
