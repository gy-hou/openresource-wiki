# 论文精读 Prompt

> 💡 **适用场景**：结构化精读论文并提炼可执行结论
>
> **推荐模型**：Claude Opus（长上下文）
>
> **难度**：⭐⭐ 进阶

## Prompt 正文

```text
你是一位研究助理，请对我提供的论文内容做结构化精读。

请严格按以下结构输出：
1. 一句话总结（不超过 40 字）
2. 核心贡献（3-5 条）
3. 方法论拆解（问题定义、方法框架、关键技术点）
4. 实验结果（数据集、指标、主要结果、对比基线）
5. 局限性（作者提到的 + 你推断的）
6. 与其他工作的关系（与经典方法/同期方法对比）
7. 可复现建议（实现难点、可能的复现路径）

输出要求：
- 使用中文
- 用小标题分节
- 对不确定信息明确标注“论文未明确说明”

论文内容如下：
[在这里粘贴论文摘要/正文/链接]
```

!!! tip "使用技巧"
    - 提供论文 PDF 全文比只给摘要效果更好
    - 可以追加“请输出为表格”提升可比较性
    - 多篇论文可让模型先做同构字段抽取

## 使用说明

1. 先贴摘要做快速判断。
2. 再贴全文做深度分析。
3. 最后让模型输出复现清单。

## 变体版本

=== "快速扫读版"

    ```text
    仅输出：一句话总结 + 核心贡献 + 是否值得精读。
    ```

=== "深度精读版"

    ```text
    输出完整结构，并补充潜在偏差与后续研究问题。
    ```

=== "复现评估版"

    ```text
    从工程实现角度评估复现成本、风险点和时间预算。
    ```
