# 搭建个人 Wiki 的工具清单

> 分层梳理从写作到部署的关键工具

> 实践结论（2026-04）：如果目标是“快速上线 + 可持续维护”，优先 `MkDocs + GitHub Pages + VS Code`；需要更强交互再升级到前端框架。

## 框架层

| 工具 | 定位 | 优势 | 适用场景 |
|------|------|------|----------|
| MkDocs | 文档站点生成器 | 上手快、生态成熟 | 技术文档 / 个人知识库 |
| Docusaurus | React 文档框架 | 可扩展性强 | 文档 + 产品官网 |
| VitePress | Vue 文档框架 | 构建快、主题灵活 | 前端生态项目 |
| Hugo | 静态站点生成器 | 速度快、模板多 | 大体量内容站点 |

## 编辑层

| 工具 | 定位 | 优势 | 适用场景 |
|------|------|------|----------|
| VS Code | 通用编辑器 | 插件生态丰富 | 技术写作 + 开发一体 |
| Obsidian | 知识管理 | 双链与图谱能力强 | 个人知识沉淀 |
| Typora | Markdown 编辑 | 所见即所得 | 纯写作场景 |

## 部署层

| 平台 | 定位 | 优势 | 适用场景 |
|------|------|------|----------|
| GitHub Pages | 静态托管 | 免费、与 GitHub 集成 | 开源项目文档 |
| Vercel | 前端部署平台 | 自动预览、CDN 友好 | 需要快速迭代 |
| Netlify | 静态部署平台 | 配置简单、插件多 | 多站点管理 |

## 辅助层

| 类别 | 工具示例 | 作用 |
|------|----------|------|
| 搜索插件 | Material Search / Algolia | 提升站内检索体验 |
| 评论系统 | Giscus / Utterances | 构建读者反馈回路 |
| 分析工具 | Plausible / GA4 | 追踪内容表现与访问来源 |
